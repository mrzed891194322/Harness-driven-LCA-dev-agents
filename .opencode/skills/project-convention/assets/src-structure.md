# src 内部目录规范

> **上下文提示**：本文件约 6KB。阅读时应按需关注对应章节，不要一次性读取全部内容。如需了解项目整体结构（根目录、共享数据等），请另读同目录下的 `global-structure.md`。
>
> 本文件由 `project-convention` skill 维护，聚焦**模型训练项目代码（src/）内部**的目录结构、模块职责与关键约束。

---

## 1. 目录结构

```
src/
│
├── train/                  # 训练代码（训练脚本、数据处理）
├── eval/                   # 评估工具（可复用指标与验收规则）
├── model/                  # 模型实现（按模型族分目录）
├── harness/                # 领域规则与约束工具（物理/化学/环境等）
├── config/                 # 配置目录（存放模型训练的各项参数）
├── history/                # 训练历史归档，按 run1、run2... 分轮次
├── output/                 # 最终确定的交付物（最佳模型、最终报告）
├── data/                   # 数据预处理代码与预处理后的中间数据
└── temp/                   # 临时文件及 AI agent 临时脚本目录
```

## 2. 各目录/文件详细约定

### 2.1 `train/` — 训练代码

- **职责**：项目的主训练入口，包含数据构造、特征工程、模型训练、模型选择、评估和产物输出。
- **组织方式**：
  - `__init__.py`：模块 docstring。
  - `train.py`：一站式训练脚本，负责完整流水线。

#### `train.py` 典型结构

| 阶段 | 关键函数 | 说明 |
|------|----------|------|
| 数据构造 | `build_daily_dataset()` | 读取根目录 `data/` 下各数据源 CSV，清洗、对齐日期、合并为日级宽表 |
| 特征工程 | 在 `build_daily_dataset()` 内 | 时间特征（月/季/周几/年末日）、比值特征、滞后特征（lag 1/2/7）、滚动统计（窗口 3/7/14） |
| 数据划分 | 时间序列按比例截断 | Train/Valid/Test 按固定日期切分（如 1-9月/10-11月/11月中旬-12月） |
| 模型构造 | `make_baseline_models()` / `make_sklearn_models()` / `make_gbdt_models()` | 从 `model/` 各子目录导入工厂函数 |
| 训练与选择 | 遍历所有模型候选 | 在验证集上按 MAE 选择最优，有业务偏好（简单模型优先） |
| 评估 | 导入 `eval/regression.py` | 回归指标、高消耗事件 P/R/F1、月度总误差、验收通过性 |
| 历史归档 | 写入 `history/runN/` | 模型参数、模型代码快照、模型文件、预测结果、指标表、特征重要性、运行摘要 |
| 文档输出 | 写入 `history/runN/README.md` 与 `history/runN/docs/` | 本轮训练说明、数据预处理、模型选择、harness 方案、eval 问题诊断 |

- **关键约束**：
  - 目标泄漏防护：T+1 预测任务中，特征只能使用 T 日及之前的数据。
  - 时间顺序：严禁随机打乱——训练/验证/测试必须按时间顺序划分。

### 2.2 `eval/` — 评估工具

- **职责**：提供可复用的模型评估函数，与具体训练流水线解耦。
- **典型内容**：
  - `regression.py`：回归指标（MAE/RMSE/R²/sMAPE）、高消耗事件检测（P90 阈值）、月度总误差、验收规则（`AcceptanceRule` 数据类）。
- **设计原则**：函数式、无副作用、可被 `train/train.py` 或 Jupyter 分析 Notebook 复用。
- **关键约束**：评估工具不持有模型状态，只接收预测值 and 真实值。

### 2.3 `model/` — 模型实现

- **职责**：按模型族组织模型定义，每个子目录按模型类型或算法族命名（如 `baseline_models/`、`sklearn_models/`、`gbdt_models/`、`deep_learning/` 等）。
- **组织方式**：每个子目录为一个 Python 包，导出对应的工厂函数供 `train/train.py` 调用。
- **关键约束**：
  - 模型与流水线解耦：模型定义只关心架构和接口，不处理数据构造或训练流程。
  - 工厂函数返回 `dict[str, Estimator]`，键为模型名，值为可调用对象。
  - 可选依赖使用 `try/except` 导入，缺失时静默跳过。

### 2.4 `harness/` — 领域规则与约束

- **职责**：存放物理、化学、环境工程等领域的规则、约束条件和工具，在模型训练、推理或后处理时按需调用，保证模型输出符合领域规范和工程可行性。
- **内容形式**：**必须**为 Python 脚本。
- **典型内容**：
  - 物理约束（质量守恒、能量守恒、物料平衡等）。
  - 化学约束（反应方程式配平、元素守恒、浓度范围等）。
  - 环境工程规范（排放标准限值、工艺参数范围等）。
  - 自定义损失函数或正则化项（基于领域规则的 penalty）。
- **组织方式**：按领域分子目录（如 `physics/`、`chemistry/`、`environment/`）。
- **关键约束**：
  - 规则工具应为纯函数，不持有模型状态。
  - 与 `train/` 和 `model/` 解耦，通过函数调用集成。
  - 规则说明文档应注明其领域依据和参考来源。

### 2.5 `config/` — 配置目录

- **职责**：存放模型训练的各项参数、配置文件（如超参数、特征清单、实验配置、数据路径等）。
- **内容形式**：通常为 JSON、YAML、Python 配置文件或环境变量模板。
- **设计原则**：将超参数、数据源路径等从代码中抽离，方便快速调参和多实验管理。

### 2.6 `temp/` — 临时文件目录

- **职责**：存放运行时的临时文件（如中间缓存），同时**必须**用于存放 AI agent 操作项目时编写的临时脚本。这些脚本**绝不能**出现在 `temp/` 以外的任何地方。
- **关键约束**：不应纳入版本控制。

### 2.7 `data/` — 数据预处理目录

- **职责**：存放数据预处理相关的代码（如数据清洗、特征提取脚本），以及预处理后生成的中间数据文件。
- **注意**：项目的原始数据统一放在根目录的 `data/` 目录下，不要放在这里。

### 2.8 `history/` 与 `output/` — 归档与交付物

关于训练历史归档（`history/`）和最终交付物（`output/`）的详细目录结构、职责与关键约束，请直接加载并参考 `output-specification` 技能中的相关规范文档。

## 3. 模块导入约定

- **项目内导入**：使用绝对导入，以 `src` 下的模块作为起点。
  - 当前 `train.py` 中通过 `sys.path` 动态添加项目路径：
    ```python
    _PROJ = Path(__file__).resolve().parents[1]
    sys.path.insert(0, str(_PROJ))
    ```
  - 更好的方式是将项目根添加到 `pyproject.toml` 的 Python 路径中。
- **跨文件导入**：
  - `train/train.py` 从 `model/` 各子目录导入工厂函数。
  - `train/train.py` 从 `eval/` 导入评估函数。
  - `model/` 内各子目录互相独立，不交叉引用。

## 4. 新增项目的推荐流程

1. 在 `src/` 下维护核心代码。
2. 遵循 `train/`、`data`、`eval/`、`model/`、`harness/`、`config/`、`history/`、`output/` 的结构。
3. 每个 Python 包添加 `__init__.py`（可只含 docstring）。
4. 将原始数据放入根目录的 `data/`，编写提取脚本 and README。
5. 编写 `train/train.py` 作为训练入口。
6. 将可复用评估逻辑放入 `eval/`。
7. 按模型族创建 `model/<family>/` 子目录。
8. 在根目录 `plan/` 下编写项目方案文档。
9. 在项目级 `.gitignore` 中配置排除项。
