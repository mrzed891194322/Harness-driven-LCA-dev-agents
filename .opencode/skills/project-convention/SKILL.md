# project-convention

本项目的目录结构、文件位置约定和工具使用指南。

**使用方式：按需加载，非自动执行。**
当 agent 遇到以下情况时，应加载本 skill 查阅：
- 不确定新建文件应放在哪个目录
- 不清楚某目录的约定用途
- 需要了解项目推荐的读写方式
- 需要确认命名规范或代码风格

## 核心约定

- **所有输出使用中文**，包括代码注释和文档。
- **Python 环境**：默认使用 `.venv`，依赖管理优先使用 `uv`。
- **ML 项目入口**：环境工程 AI 模型训练任务统一规划在 `src/` 下。
- **参数配置**：模型训练的各项参数统一存放在 `src/config/` 下。
- **计划文档**：统一放在根目录的 `plan/` 下。
- **训练历史产物**：统一输出到 `src/history/runN/` 下，其中 `N` 为从 1 开始递增的循环轮数。

## 上下文消耗控制（必读）

- **只有本文件（SKILL.md）会在加载 skill 时自动注入上下文**（约 40 行）。
- **`assets/` 下的其他文件不会自动加载**，需 agent 显式使用 `Read` 工具读取。
- **禁止一次性读取全部参考文件**。应按需读取当前实现所需的单个文件。

## 目录规范速查

加载本 skill 后，**按需读取**以下文件（不要全读）：

| 文件 | 适用场景 | 内容 | 预估大小 |
|------|----------|------|----------|
| `assets/global-structure.md` | 主 agent、全局规划 | 根目录结构、`src/`、`data/`、工具与环境约定、Git 规范 | ~6KB |
| `assets/src-structure.md` | 编码 agent、src 实现 | src 内部目录结构、模块职责、关键约束 | ~6KB |

## 何时不使用

- 若用户已明确要求文件放在特定位置，遵循用户指令。
- 若当前任务与项目结构无关（如纯算法讨论），无需加载。
