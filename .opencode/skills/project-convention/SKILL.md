# project-convention

本项目的目录结构、文件位置约定和工具使用指南。此项目旨在使用多智能体在 harness 下进行合规化的 LCA (Life Cycle Assessment) 输出。

**使用方式：按需加载，非自动执行。**
当 agent 遇到以下情况时，应加载本 skill 查阅：
- 不确定新建文件应放在哪个目录
- 不清楚某目录的约定用途
- 需要了解项目推荐的读写方式
- 需要确认命名规范或代码风格

## 核心约定

- **语言自动适应**：根据用户使用的语言决定后续回复语言（遵循 `AGENT.md`）。
- **Python 环境**：默认使用 `.venv`，依赖管理优先使用 `uv`。安装新包需事先征得用户同意。
- **核心代码区**：项目核心处理代码与动态状态规划在 `src/` 下。
- **输入与输出**：外部数据、规划放入 `input/` 下；最终交付物和导出结果存放于 `output/` 下。
- **文档与知识库**：位于 `docs/` 下，包括项目说明文档与基础静态知识库。

## 上下文消耗控制（必读）

- **只有本文件（SKILL.md）会在加载 skill 时自动注入上下文**。
- **`assets/` 下的其他文件不会自动加载**，需 agent 显式使用工具读取。
- **禁止一次性读取全部参考文件**。应按需读取当前实现所需的单个文件。

## 目录规范速查

加载本 skill 后，**按需读取**以下文件（不要全读）：

| 文件 | 适用场景 | 内容 | 预估大小 |
|------|----------|------|----------|
| `assets/global-structure.md` | 主 agent、全局规划 | 根目录结构、`input/`、`output/`、`docs/`、工具与环境约定 | ~4KB |
| `assets/src-structure.md` | 编码 agent、src 实现 | src 内部目录结构（`eval/`, `history/`, `knowledge/`, `plan/`, `tmp/`）、模块职责 | ~4KB |

## 何时不使用

- 若用户已明确要求文件放在特定位置，遵循用户指令。
- 若当前任务与项目结构无关（如纯概念讨论），无需加载。
