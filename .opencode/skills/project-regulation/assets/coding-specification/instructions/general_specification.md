# 通用操作与命令行执行规范 (General Operations Specification)

本文件由 `project-regulation` 技能维护，定义了在本项目中执行各类环境操作、文件读写、以及命令行（如 Bash/PowerShell）执行时的安全边界。

## 核心规范

为了确保系统的安全性与操作的合规性，所有 Agent 在执行环境操作时，必须严格遵守以下约定：

1. **项目内作用域限制**
   - 所有文件读写、代码编写及命令执行（包括 Bash/PowerShell 命令行操作等）的边界**必须严格限制在本项目目录内**。
   - 严禁读取、修改、创建或删除项目外部的任何文件，或对项目外部的文件/路径执行任何操作。

2. **例外场景（外部工具）**
   - 只有在明确加载并使用 `external-tools` 技能（例如调用 openLCA 软件或进行外部 RAG 数据库检索等）且获得用户授权时，方可与约定的外部系统或服务进行交互。
   - 即使在使用 `external-tools` 时，也必须遵循该技能规定的安全通道与边界，严禁进行任意的外部文件编辑。
