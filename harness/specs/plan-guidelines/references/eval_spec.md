# LCA 项目计划交付验收与自检规范

本文件用于检查生成或修订后的 `execution_plan.md` 和 `todo_list.md` 是否符合计划阶段交付标准。

## 1. 文件与模板自检

- 是否在 `workspace/plan/` 下生成或更新了 `execution_plan.md`。
- 是否在需要时生成或更新了 `todo_list.md`。
- `execution_plan.md` 是否严格遵守 `harness/specs/plan-guidelines/references/templates/execution_plan.md`。
- `todo_list.md` 是否严格遵守 `harness/specs/plan-guidelines/references/templates/todo_list.md`。
- 两个文件是否保留模板规定的 YAML front matter、标题层级和关键字段结构。

## 2. Execution Plan 内容自检

- 范围定义：是否明确研究对象、目的、系统边界、截断规则、功能单位和分配原则。
- LCIA 方法：是否明确选用方法和关注指标；无法确认时是否进入 Todo。
- 数据与环境：是否说明 openLCA 连接、数据库、已有参考资料和数据质量。
- 模型方案：是否清晰列出拟创建的 Flow、Process、Product System 和背景数据链接。
- 结果解释：是否包含完整性检查、热点分析、可视化和不确定性/敏感性分析方案（如适用）。
- 来源追溯：关键数据、假设和不确定项是否可追溯。

## 3. Todo List 内容自检

- `todo_list.md` 是否仅包含两个一级标题：
  - `## 1. 待完善清单内容与反馈答复`
  - `## 2. 用户额外补充意见与文字需求`
- 是否严禁出现任何其他一级标题，例如 `## 3.`。
- 第一部分中的所有清单项是否完全、逐条对应 `execution_plan.md` 中留空、占位符、数据缺口或待决策事项。
- 每个清单项是否包含用户反馈/决议答复段落和 `***✍️ 用户填写内容区***`。
- 是否不存在与执行计划问题无关的编造项或赘余项。

## 4. 修改任务自检

当任务是修改既有计划时，必须检查：

- 是否读取并保留了仍有效的原计划内容。
- 是否只针对用户反馈或明确修改需求进行增量修正。
- 是否把已解决的 Todo 内容同步固化进 `execution_plan.md`。
- 是否更新或标记了 `todo_list.md` 中对应条目。
- 是否避免无关重写、无关删除或重新生成整份计划。

## 5. 交付判定

- 如任一检查项不满足，必须判定为“需要改进”，并给出可执行修改建议。
- 如因缺乏信息无法完善某一部分，但已在 Todo 中详细记录供人类决策，可视为该项达标。
- 自检未通过时不得宣称完成。
