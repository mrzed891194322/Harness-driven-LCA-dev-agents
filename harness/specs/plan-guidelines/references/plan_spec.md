# LCA 项目计划内容与交付规范

本文件规定 LCA 执行计划与 Todo List 必须满足的内容、结构和交付要求。执行步骤不在本文件中定义。

## 1. 必须交付的文件

所有计划阶段产物必须集中放置于 `workspace/plan/`。

- `workspace/plan/execution_plan.md`：LCA 项目执行计划。
- `workspace/plan/todo_list.md`：待完善清单与用户反馈答复文档。

## 2. 执行计划核心内容

`execution_plan.md` 必须包括并体现以下内容。

### 2.1 基本范围定义

- 研究对象与目的：明确评估的产品、技术或系统，阐述开展研究的目的。
- 功能单位 (Functional Unit, FU)：设定明确的功能单位，包含基准流量和物理单位，并阐明设定依据。
- 系统边界与截断规则：界定生命周期边界，说明质量/能量截断比例及特殊排除。
- 多产出分配原则：若存在副产品，说明分配优先级。
- 生命周期影响评价方法：确定 LCIA 方法与关注的核心指标。

### 2.2 数据与环境基础

- openLCA 连接地址、端口和连接状态。
- 已载入或计划使用的数据库。
- 已有物料清单、技术文档、实测数据等参考资料。
- 核心数据质量、完整性和代表性的初步判断。
- 对现有 openLCA 数据库和 LCIA 方法的参考情况；无法确认时必须列入 Todo。

### 2.3 模型细节方案

- 需要在 openLCA 中新建的 Flow。
- 需要在 openLCA 中新建的 Process。
- 背景数据链接方案与选择理由。
- Product System 构建方案。
- 是否需要 Project 功能进行多情景对比分析。

### 2.4 结果验证与解释方案

- 完整性与一致性检查方案。
- 热点分析方案，例如贡献树、桑基图或其他可视化。
- 不确定性分析或敏感性分析方案（如适用）。

## 3. Todo List 内容规范

`todo_list.md` 必须严格且仅包含以下两个一级标题：

1. `## 1. 待完善清单内容与反馈答复`
2. `## 2. 用户额外补充意见与文字需求`

第一部分中的所有清单项必须完全、逐条参考并对应 `execution_plan.md` 中留空、带中括号占位符、存在不确定性、缺口或待决策的问题。

每个动态生成的清单项末尾必须包含结构一致的用户反馈/决议答复段落及 `***✍️ 用户填写内容区***` 输入区域。

严禁生成 `## 3.` 或更多一级标题；严禁编造或插入与 `execution_plan.md` 中提出的问题无关的赘余内容。

## 4. 模板要求

- `execution_plan.md` 必须遵守 `harness/specs/plan-guidelines/references/templates/execution_plan.md`。
- `todo_list.md` 必须遵守 `harness/specs/plan-guidelines/references/templates/todo_list.md`。
- 输出文件必须保留模板规定的 YAML front matter、一级/二级标题和关键字段结构。

## 5. 信息不确定性处理

- 不得把未经证实的信息写成确定事实。
- 缺失数据、边界疑问、分配决策、背景库匹配问题和 LCIA 方法选择疑问必须进入 Todo。
- 已由用户反馈解决的问题必须同步固化到执行计划，并更新 Todo 状态。
