# PET 水瓶 Case 1/2/3 LCA 执行计划

## 1. 给 Agent 的任务

为 Harness LCA Agent 执行一个教学用途的端到销售点（cradle-to-point-of-sale）比较案例：比较三条供应路线向德国销售点交付 1,000 个 1 L PET 静水瓶时的潜在环境影响。

你必须完成计划、LCI 设计、Provider 查询与消歧、受控导入、模型图验证、LCIA、解释和运行记录。任何阶段缺少证据或需要人工决定时，保留 `unresolved` 状态，不得猜测 UUID、Provider、地域、方法或计算结果。

本任务用于开发回归，不支持公开产品优越性声明，也不构成 ISO 认证或关键审查。

## 2. 研究目标与用途

- **研究问题**：在固定瓶体质量、灌装量和前景建模假设下，三种粒料来源与运输路线如何影响 LCIA 结果和热点构成？
- **预期用途**：验证项目的 `计划 → LCI → openLCA → 评估` 工作流，并形成可复现的开发基线。
- **预期受众**：项目开发者、测试人员和内部 LCA 审核者。
- **比较声明**：不用于拟向公众披露的比较声明。若用途发生变化，立即停止发布结论并升级到独立专家及适用的评审小组流程。

## 3. 功能、功能单位与参考流

- **功能**：在德国销售点提供以 PET 瓶、HDPE 盖和 PP 标签包装的 1 L 静水。
- **功能单位**：在德国销售点交付 **1,000 个 1 L PET 静水瓶**。
- **参考流**：销售点交付的灌装水瓶 1,000 item，总产品质量 1,065 kg。
- **比较基础**：三个情景使用相同功能、功能单位、参考流、瓶型、灌装量和前景过程结构。

所有 LCI 和 LCIA 结果必须归一化到此功能单位。若 openLCA 产品系统的定量参考流不是 1,000 item，必须在计算设置中明确缩放并验证等价性。

## 4. 固定案例事实

### 4.1 物料

| 组件 | 单瓶 | 功能单位 | 约束 |
| :--- | ---: | ---: | :--- |
| PET 瓶体 | 0.060 kg | 60 kg | bottle-grade PET granulate |
| HDPE 瓶盖 | 0.004 kg | 4 kg | HDPE granulate |
| PP 标签 | 0.001 kg | 1 kg | PP granulate |
| 静水 | 1.000 kg | 1,000 kg | 德国/欧洲供水代理 |
| 空瓶组件合计 | 0.065 kg | 65 kg | 质量检查点 |
| 灌装产品合计 | 1.065 kg | 1,065 kg | 质量检查点 |

前景模型假定粒料到塑料组件的质量收率为 100%，没有生产废料。这是教学案例假设，不是工厂事实。

### 4.2 运输

货运活动统一按 `货物质量（t） × 距离（km） = t·km` 计算。

| 情景 | 运输段 | 地域/方式 | 推导 | 固定需求 |
| :--- | :--- | :--- | :--- | ---: |
| Case 1 | A：粒料至组件生产 | Europe 卡车 | 0.065 t × 200 km | 13 t·km |
| Case 1 | B：空组件至灌装 | Europe 卡车 | 0.065 t × 200 km | 13 t·km |
| Case 1 | C：灌装瓶至 POS | Europe 卡车 | 1.065 t × 50 km | 53.25 t·km |
| **Case 1 合计** | A+B+C | **Europe 卡车** | 13 + 13 + 53.25 | **79.25 t·km** |
| Case 2 | A：粒料至组件生产 | China 铁路 | 0.065 t × 5,000 km | **325 t·km** |
| Case 2 | B：空组件至灌装 | Europe 卡车 | 0.065 t × 200 km | 13 t·km |
| Case 2 | C：灌装瓶至 POS | Europe 卡车 | 1.065 t × 2,600 km | 2,769 t·km |
| **Case 2 卡车合计** | B+C | **Europe 卡车** | 13 + 2,769 | **2,782 t·km** |
| Case 3 | A1：粒料陆路段 | RoW 卡车 | 0.065 t × 300 km | **19.5 t·km** |
| Case 3 | A2：粒料海运段 | Global 海运 | 0.065 t × 13,887 km | **902.655 t·km** |
| Case 3 | B+C | Europe 卡车 | 与 Case 2 相同 | **2,782 t·km** |

不得把 `kg·km` 数值直接填入以 `t·km` 为参考单位的交换。若所选 flow 的参考单位是 `kg·km`，应使用数值放大 1,000 倍并在换算记录中说明。

## 5. 来源与已解决冲突

案例事实来自[本地水瓶教程](<../../../harness/knowledge/inputs/user_file/水瓶案例学习/水瓶案例学习.md>)：

- 教学性质与旧数据库版本：第 37–43 行；
- 情景与供应路线：第 45–76 行；
- 目标、功能单位和边界：第 78–189 行；
- 运输距离、物料质量、100% 收率和生产能耗缺口：第 191–207 行。

教程第 197 行表格给 Case 3 的 RoW 卡车距离为 300 km，第 199 行正文却写为 200 km。本基准显式采用表格的 300 km，使固定需求为 19.5 t·km；不得静默改用 13 t·km。教程使用 openLCA 1.4.1 beta6、ecoinvent 3.1 allocation default 和 CML 2001，故其名称、UUID 与 LCIA 数值只作为历史教学上下文，不能直接移植到当前环境。

## 6. 系统边界

### 6.1 纳入

- PET、HDPE 和 PP 粒料的背景供应链；
- 自来水的背景供应链；
- 粒料、空瓶组件和灌装瓶的固定货运活动；
- 前景中的粒料供应、塑料组件生产、灌装和销售点运输；
- 背景数据集随锁定 `Cutoff, S` system model 内含的上游过程。

### 6.2 排除或暂缺

- 塑料成型、贴标、封盖和灌装设备的前景电力、热力与辅助材料；
- 生产损耗和废料处理；
- 二次/三级包装、仓储、销售点运营与制冷；
- 消费者前往销售点、使用阶段、押金返还、回收和最终处置；
- 工厂、车辆和设备等资本品在前景中的额外建模；背景数据集既有边界不另行修改。

这些排除项必须出现在结果限制中。特别是生产能耗缺失不能被描述为“已证明不重要”；只有敏感性分析支持时，才能限定性地说明其对本案例排序的影响。

### 6.3 截断与分配

- 不另设“低于 1% 自动删除”的隐式截断规则；任何遗漏按上述清单逐项记录原因和可能影响。
- 前景过程按单一参考产品设计，不预期发生共同产品分配。
- 若选定背景数据集含分配，其逻辑由锁定的 ecoinvent `Cutoff, S` 模型决定，并在报告中披露。
- 若实际建模出现共同产品或废物，停止并记录人工决策点，不得临时选择质量或经济分配。

## 7. 前景过程分解与预期拓扑

每个情景建立语义相同、参数不同的四段模型：

```text
PET/HDPE/PP 背景 Provider
          ↓
      粒料供应
          ↓  65 kg 粒料组合
  塑料组件生产 + 运输 A
          ↓  1,000 套空瓶组件（65 kg）
       灌装 + 水 + 运输 B
          ↓  1,000 个灌装瓶（1,065 kg）
   销售点运输 + 运输 C
          ↓
德国 POS 的 1,000 个 1 L 水瓶
```

建模实现可使用“1,000 套/个”为中间参考流，也可使用参数化单瓶过程再按 1,000 缩放；选择必须在三个情景中一致，且中间 flow ID、交换方向和 Provider 连接可从模型图验证。

建议将运输 A 放在塑料组件生产阶段、运输 B 放在灌装阶段、运输 C 放在销售点运输阶段，以对应教程的运输分组。粒料供应过程用于显式承载三种树脂的背景 Provider，不得捏造一个物理上未发生的“混合生产”过程。

## 8. Provider 与地域要求

### 8.1 情景语义

| 数据对象 | Case 1 | Case 2 | Case 3 |
| :--- | :--- | :--- | :--- |
| PET/HDPE/PP 背景地域 | RER/Europe | RoW（中国生产语义的教程代理） | RoW（中国生产语义的教程代理） |
| 粒料运输 | Europe lorry | China train | RoW lorry + Global ship |
| 空组件和灌装瓶运输 | Europe lorry | Europe lorry | Europe lorry |
| 水 | Germany 优先，Europe 代理次之 | 同 Case 1 | 同 Case 1 |

教程用 RoW 表示 Case 2/3 的粒料背景数据，但叙事地点为中国。必须把这项地理代理写入限制。若当前数据库有合适的 CN 粒料数据，可作为敏感性方案，不得在主情景中静默替换 RoW 并仍声称复现教程事实。

### 8.2 查询与消歧

对每个背景输入执行只读描述符查询，并保存：实体类型、查询词、返回总数、完整名称、UUID、参考产品/流、地域、system model、选择理由和查询时间。

当前环境存在多个名称完全相同但 UUID 不同的 PET、HDPE、运输或供水过程。名称匹配不等于地域匹配。只有在进一步读取或由合格人员确认地域后才能选择；无法确认时标记 `unresolved_provider` 并停止导入该情景。

禁止：

- 从教程、记忆或另一个数据库版本复制 UUID；
- 因搜索结果第一项“看起来合理”就选为 Provider；
- 用生产数据集、市场数据集或 market group 相互替换而不说明；
- 为了让 Product System 连起来而连接参考产品不一致的过程。

## 9. LCIA 设置

- **背景数据库基线**：ecoinvent 3.11，`Cutoff, S`；运行时再次验证。
- **LCIA 方法**：`CML v4.8 2016 no LT`；保存方法 UUID，不得误选同库中的 `CML v4.8 2016`。
- **必报内容**：未加权的完整 characterization 结果；至少单列气候变化、化石资源耗竭、酸化、富营养化、人类毒性和臭氧层消耗中方法实际提供的对应类别。
- **可选元素**：本基准不使用归一化、分组或加权来得出总体优劣。
- **计算一致性**：三个 Product System 使用相同分配设置、参数、参考流和方法。

LCIA 是相对于功能单位的潜在影响指标。报告不得把结果写成对类别终点、阈值、安全边际、风险或实际损害的预测。

## 10. 数据缺口与敏感性分析

至少执行并记录以下敏感性检查；若因数据不足不能运行，标记 `needs_human_review` 或 `conditional_pass`，不得删除该要求：

1. **前景生产能耗**：为成型、贴标、封盖与灌装加入一个有来源的能耗代理区间，检查热点和情景排序是否改变。
2. **运输距离**：对主要长距离运输统一执行 ±20% 变化，检查各影响类别差异是否稳定。
3. **地域 Provider**：比较主情景 RoW 代理与经验证的 CN 或其他合理地域数据集，若可用。
4. **Case 3 来源冲突**：将 200 km（13 t·km）作为冲突来源敏感性，与主情景 300 km（19.5 t·km）比较。
5. **影响类别完整性**：不得只以气候变化类别代表总体环境表现；逐类别检查排序与权衡。

水的 Provider、塑料成型能耗、100% 收率、生产地点代表性、终端阶段排除和数据库版本差异均列入数据缺口清单。

## 11. 执行顺序与验证门

1. **计划**：生成 execution plan、todo、来源与决策表；人工确认用途不是公开比较。
2. **LCI 设计**：生成前景 Flow、Process、Product System 和映射报告；确定性检查物料、运输和引用。
3. **Provider 查询**：逐项保存候选与消歧证据；所有 UUID 均来自当前环境查询。
4. **导入前验证**：检查 schema、定量参考流、交换方向、单位、重复 ID 和写入范围。
5. **导入**：记录成功/失败计数与实体 UUID；部分失败即停止后续计算。
6. **模型图**：读取三个 Product System 的节点和边，对照第 7 节拓扑；无孤立输入。
7. **LCIA**：保存计算设置和原始结果；成功返回但无类别结果时判定 `calculation` 失败。
8. **解释**：执行热点、完整性、敏感性和一致性检查；结论与限制回扣目标和用途。
9. **记录与审核**：填写 [benchmark_run_record_template.md](benchmark_run_record_template.md)，再按 [lca_quality_evaluation.md](lca_quality_evaluation.md) 评价。

只有实际证据通过相应检查后才能宣称该阶段完成。Agent 自述、文件存在或命令退出码为 0 都不是单独的完成证据。

- 导入报告、导入后实体清单和输入/输出哈希；

- 一份填写完整的 benchmark run record。

教程 ecoinvent 3.1 的 LCIA 数值不得作为当前环境严格金标。第一次通过审核的 ecoinvent 3.11 运行只能成为“锁定环境数值基线”，并须附数据库、Provider、方法、哈希和复审人。
