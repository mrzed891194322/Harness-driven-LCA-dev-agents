# LCA 质量评价方法：ISO 硬性门禁与工程证据等级

## 1. 目的与边界

本方法用于评价 LCA 产物是否覆盖适用的 ISO 14040/14044 要求，以及证据是否足以支持项目的开发回归结论。它采用两层结构：

1. **ISO 硬性要求门禁**：逐项判断适用性并检查 ISO 14044 中的规范性要求；
2. **工程质量等级**：用项目自定义的 0–3 级描述证据成熟度。

ISO 14040/14044 没有给出适用于所有 LCA 的通用百分制。本文件的证据等级、状态名称和判定流程是项目治理方法，不是 ISO 分数、认证规则或 ISO 官方合格评价。ISO 14044 也明确其本身不用于注册或认证。

本方法不能替代合格 LCA 专家的专业判断、第三方报告责任或公开比较声明所需的独立关键审查。

## 2. 规范依据与本地定位

主要依据是本地原文：

- [ISO 14040:2006](<../../../harness/knowledge/inputs/static_ref/standards/ISO14040-2006/ISO 14040-2006_.md>)：说明 LCA 的原则与框架；其第 696–698 行要求开展 LCA 时应用 ISO 14044 的要求。
- [ISO 14044:2006](<../../../harness/knowledge/inputs/static_ref/standards/ISO14044-2006/ISO 14044-2006_.md>)：规定 LCA 的要求与指南。

本文使用的主要 ISO 14044 定位为：

| 主题 | 本地原文行号 | 主要内容 |
| :--- | :--- | :--- |
| 目标与范围 | 500–590 | 四阶段、用途、原因、受众、比较声明、功能单位、参考流、边界和截断 |
| LCI 计算与验证 | 690–810 | 计算记录、有效性、功能单位归一化、边界修订和分配 |
| LCIA | 850–1095 | 必需元素、类别/指标/模型、来源、分类、表征、可选元素和比较限制 |
| 解释 | 1095–1255 | 重大问题、完整性、敏感性、一致性、结论和限制 |
| 报告与关键审查 | 1260–1425 | 透明报告、第三方报告、公开比较附加要求和评审小组 |

执行评价时须确认适用标准版本及修正案。条款有变化时，以适用的最新受控原文为准，并记录影响分析；不得只依赖本文摘要。

## 3. 评价原则

- **先判断适用性，再判通过**：例如 LCIA 条款不适用于纯 LCI 研究，但公开比较用途会触发额外要求。
- **逐项追踪规范性措辞**：硬门禁只把适用的 `shall` 要求作为强制项；`should` 建议和说明性内容不伪装成强制要求。
- **证据优先**：Agent 自述、文件存在或工具成功退出不能独立证明要求已满足。
- **不以平均值抵消失败**：任一适用硬门禁缺失即 `fail`，其他维度高分不能补偿。
- **与预期用途相称**：内部教学回归、第三方报告和拟向公众披露的比较声明需要不同深度的证据与审查。
- **相对结果**：LCIA 结果相对于功能单位，不是对类别终点、阈值、安全边际、风险或实际损害的预测。

## 4. 第一步：适用性声明

在检查前填写：

| 问题 | 结论 | 证据/理由 |
| :--- | :--- | :--- |
| 是完整 LCA 还是仅 LCI 研究 | `{LCA/LCI}` | `{path}` |
| 预期用途与受众 | `{value}` | `{path}` |
| 是否向第三方沟通 | `{yes/no}` | `{path}` |
| 是否支持拟向公众披露的比较声明 | `{yes/no/unresolved}` | `{path}` |
| 是否使用 LCIA 可选元素 | `{normalization/grouping/weighting/none}` | `{path}` |
| 关键审查类型 | `{none/expert/panel/unresolved}` | `{path}` |
| 适用标准与修正案 | `{versions}` | `{path}` |

`unresolved` 的公开比较状态本身需要人工判断。在用途确认前，不得发布环境优越性或等效性结论。

## 5. 第二步：ISO 硬性要求门禁

以下清单是评测路由，不替代原文。每项记录 `applicable`、`not_applicable`（附理由）、`pass`、`fail` 或 `needs_human_review`，并给出证据位置。

### 5.1 总体、目标与范围

依据：ISO 14044 第 500–590 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| GS-01 | 完整 LCA 包含目标与范围、LCI、LCIA 和解释四阶段 | 四阶段产物及其相互引用；纯 LCI 研究说明适用例外 |
| GS-02 | 目标与范围清楚定义、与用途一致 | 受控目标与范围文档及修订记录 |
| GS-03 | 用途、开展原因、预期受众、是否公开比较均明确 | 四项均有无歧义声明 |
| GS-04 | 产品系统和功能/性能特征明确 | 可复核的系统与功能描述 |
| GS-05 | 功能单位清楚、可测量且符合目标与范围 | 数量、质量、时间、地点或性能维度完整 |
| GS-06 | 参考流已定义；比较基于相同功能与功能单位 | 功能单位—参考流换算及系统等效性说明 |
| GS-07 | 系统边界、单位过程和详细程度与目标一致 | 边界描述、过程清单和流程图 |
| GS-08 | 省略项及其原因和影响透明 | 排除清单、显著性或敏感性证据 |
| GS-09 | 截断准则和假设明确，包含质量、能量与环境显著性考虑 | 阈值、计算和最终影响评价 |
| GS-10 | 分配程序、LCIA 方法、数据要求、假设、价值选择、限制、数据质量、评审与报告格式已描述 | 范围章节逐项覆盖；不适用项附理由 |

### 5.2 LCI 数据、计算与验证

依据：ISO 14044 第 690–810 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| LCI-01 | 数据收集覆盖与目标一致的能源、原料、辅助输入、产品、副产品、废物和环境排放 | 数据收集表、单位过程清单和缺口记录 |
| LCI-02 | 所有计算步骤及假设被明确记录并一致应用 | 可重算公式、脚本/工作表、版本和假设 |
| LCI-03 | 数据有效性检查已执行 | 质量/能量平衡、释放因子比较或等价验证及异常处置 |
| LCI-04 | 每个单位过程有适当参考流，输入输出关联到该流 | 定量参考流和单位过程换算 |
| LCI-05 | 全系统输入输出最终关联到功能单位 | 过程链缩放和归一化检查 |
| LCI-06 | 聚合程度与目标一致，未合并语义或影响不等价的数据 | 聚合规则、流映射和例外说明 |
| LCI-07 | 边界修订依据敏感性与截断准则，并保留结果 | 初始/最终边界、敏感性和变更理由 |
| LCI-08 | 分配过程被识别，程序有说明、守恒且一致应用 | 分配决策、前后平衡及替代方法敏感性；无分配时证明单一产品结构 |
| LCI-09 | 缺失或异常数据按数据质量要求处理 | 缺失值、代理值、异常和替代数据记录 |

### 5.3 LCIA

仅完整 LCA 适用。依据：ISO 14044 第 850–1095 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| IA-01 | LCIA 与目标、范围、LCI 数据质量、边界和截断协调 | LCIA 计划与充分性检查 |
| IA-02 | 包含类别/指标/表征模型选择、分类和表征三个必需元素 | 方法配置、分类和类别结果 |
| IA-03 | 类别、指标、模型及相关来源已引用，名称准确 | 方法版本、UUID、来源与类别清单 |
| IA-04 | 选择有理由、与目标和范围一致，并覆盖相关环境问题 | 选择理由和类别完整性评价 |
| IA-05 | 环境机制、模型、指标、终点、适用 LCI 结果和表征因子可识别 | 方法文档或受控引用 |
| IA-06 | 表征计算、假设和价值选择有记录 | 原始结果、方法设置和可重算记录 |
| IA-07 | 数据不足时已补充数据或修订目标与范围 | 迭代记录；不得用空结果继续解释 |
| IA-08 | 归一化、分组或加权如被使用，符合目标、透明且保留变换前结果 | 计算、参考值、价值选择和原始 profile |
| IA-09 | 报告明确 LCIA 的相对性和固有限制 | 限制章节，不将指标表述为实际风险或损害 |

### 5.4 解释

依据：ISO 14044 第 1095–1255 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| INT-01 | LCI/LCIA 结果按目标和范围解释 | 目标—结果—解释追踪 |
| INT-02 | 显著输入、输出、过程、影响类别及方法选择被识别 | 热点/贡献分析和方法问题清单 |
| INT-03 | 功能、功能单位、边界和数据质量限制在解释中复核 | 一致性和限制记录 |
| INT-04 | 完整性检查处理所有相关缺失信息并记录理由 | 完整性表与返回前阶段的迭代记录 |
| INT-05 | 敏感性检查评价数据、分配、方法和指标计算对结论的影响 | 预设情景、结果和结论稳定性 |
| INT-06 | 一致性检查覆盖数据质量、地域/时间、分配、边界和 LCIA 应用 | 跨过程/跨情景一致性矩阵 |
| INT-07 | 结论源于重大问题及评价，并符合用途、假设、数据质量和限制 | 结论链、限制和必要的回修记录 |
| INT-08 | 建议基于最终结论并与预期用途相关 | 建议—结论对应关系；无建议时说明 |

### 5.5 报告与可复核性

依据：ISO 14044 第 1260–1377 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| REP-01 | 报告类型与格式在范围阶段定义 | 目标与范围中的报告约定 |
| REP-02 | 结果和结论完整、准确、无偏并面向预期受众 | 报告审核记录 |
| REP-03 | 数据、方法、假设、限制和权衡透明，详细到足以理解和复核 | 报告、附件、evidence manifest 和运行记录 |
| REP-04 | 结果的使用方式与研究目标一致 | 用途、发布边界和限制 |
| REP-05 | 向任何第三方沟通时，已准备并提供适用的第三方报告 | 第三方报告、提供记录和第 5.2 节所需内容 |
| REP-06 | LCIA 报告包含方法、结果、类别、模型、因子、价值选择和相对性限制 | LCIA 章节及来源 |
| REP-07 | 解释报告包含结果、方法/数据限制、数据质量和专家判断透明度 | 解释章节及审核记录 |
| REP-08 | 如开展关键审查，报告包含审核者、审查报告和回应 | 审核包 |

### 5.6 拟向公众披露的比较声明

仅当适用性声明为 `yes` 时启用；为 `unresolved` 时先给 `needs_human_review`。依据：ISO 14044 第 582–590、1075–1095、1219、1379–1425 行。

| ID | 检查点 | 通过所需证据 |
| :--- | :--- | :--- |
| PUB-01 | 比较系统使用相同功能和功能单位，系统等效性得到评价 | 功能、参考流、边界、数据质量和方法等效性矩阵 |
| PUB-02 | 质量、能量与环境显著性均进入最终边界敏感性分析 | 截断与敏感性结果 |
| PUB-03 | LCIA 使用足够全面的类别指标并逐类别比较 | 完整 LCIA profile；不以单项或总分代替 |
| PUB-04 | LCIA 不是总体优越/等效声明的唯一依据 | LCI、数据质量、敏感性、不确定性和其他相关信息 |
| PUB-05 | 报告覆盖材料/能量流、精度、完整性、代表性、系统等效性和评审过程 | 公开比较附加报告章节 |
| PUB-06 | 报告说明类别指标的国际接受程度、科学技术有效性和环境相关性 | 方法依据和专业评价 |
| PUB-07 | 详细敏感性、不确定性及差异显著性评价已完成 | 分析结果与解释性陈述 |
| PUB-08 | 已由利益相关方评审小组开展关键审查 | 独立主席/成员、评审报告、意见回应和报告收录证据 |

公开比较、系统等效性、类别完整性、方法科学有效性和差异显著性通常需要合格人员判断。自动检查只能准备证据；在审核完成前状态为 `needs_human_review`，若适用材料客观缺失则为 `fail`。

## 6. 第三步：工程质量等级

对每个评测维度单独给出证据等级，不计算通用百分制：
- `0—缺失`：没有相关内容或证据；
- `1—已陈述`：有文字说明，但不可回读或验证；
- `2—有证据`：存在来源或计算记录，但覆盖不完整；
- `3—已验证`：证据完整，并通过确定性检查或合格人员复核。

| 等级 | 名称 | 判定 |
| ---: | :--- | :--- |
| 0 | 缺失 | 没有相关内容，或声称不适用但没有理由 |
| 1 | 已陈述 | 有文字说明，但来源、计算或执行状态不能回读验证 |
| 2 | 有证据 | 有可回读来源、计算或工具记录，但覆盖不完整、存在未关闭异常，或尚未复核 |
| 3 | 已验证 | 证据完整，通过确定性检查，且需要专业判断的部分已由合格人员复核 |

建议维度为：

| 维度 | 典型证据 |
| :--- | :--- |
| 目标、用途与范围 | 目标与范围文档、修订和适用性决定 |
| 功能单位、参考流与系统等效性 | 性能定义、换算、比较矩阵 |
| 系统边界、截断与分配 | 流程图、排除/分配决策、敏感性 |
| 数据质量与来源 | evidence manifest、代表性、缺失与代理记录 |
| LCI 数值与模型 | schema、换算、平衡、Provider 和模型图 |
| LCIA 方法与结果 | 方法/UUID、分类、表征、原始 profile 和限制 |
| 解释 | 热点、完整性、敏感性、一致性、不确定性和结论链 |
| 报告与可复现性 | 无偏报告、环境清单、哈希、日志和运行记录 |
| 关键审查与治理 | 审核者资格、审查报告、意见回应和发布边界 |

“已验证”不等于“ISO 认证”。例如，一个内部教学案例可以在其受限用途内达到 3 级证据，但仍不适合公开产品比较。

## 7. 总体判定规则

按顺序判定：

1. **适用的 ISO `shall` 要求客观缺失或违反**：`fail`。
2. **公开比较、关键方法选择、系统等效性、差异显著性或其他机器无法可靠判断的问题尚待合格人员决定**：`needs_human_review`。若所需材料本身缺失，仍为 `fail`。
3. **自动检查通过，但有未解决的重要数据/模型限制，且结论只能在限定条件下使用**：`conditional_pass`。必须记录使用边界、负责人和复审条件。
4. **全部适用硬门禁通过，关键维度达到 3—已验证，必要人工复核完成，且结论符合预期用途**：`pass`。

优先级为 `fail` 高于 `needs_human_review`，后者高于 `conditional_pass`，最后才是 `pass`。不得用各维度的平均等级改变这个顺序。

## 8. 评价输出格式

每次评价至少输出：

| ID | 适用性 | 硬门禁状态 | 证据等级 | 证据位置 | 发现/缺口 | 责任人 | 复审条件 |
| :--- | :--- | :--- | ---: | :--- | :--- | :--- | :--- |
| `{GS-01}` | `{applicable}` | `{pass}` | `{3}` | `{path}` | `{none}` | `{name}` | `{none}` |

并附：

- 适用性声明和标准/修正案版本；
- 所有 `fail` 与 `needs_human_review` 项，不只给汇总；
- 各工程维度的 0–3 级及理由；
- 最终四态结论、适用范围、限制、审核者和日期；
- 若比较数值基线，链接到填写完整的 [benchmark_run_record_template.md](benchmark_run_record_template.md)。

## 9. 水瓶基准的默认应用

PET 水瓶 Case 1/2/3 默认用途是内部教学与开发回归，不是拟向公众披露的比较声明。因此 PUB 门禁默认 `not_applicable`，但必须保留“不公开比较”的用途证据。其余目标与范围、LCI、LCIA、解释和报告门禁仍适用。

由于案例缺少工厂前景能耗和终端阶段数据，即使结构、连接和计算全部通过，也需要根据敏感性结果判断是 `conditional_pass` 还是 `pass`。在没有充分敏感性和合格人员复核时，不得将其提升为实际产品环境绩效结论。
