# Harness LCA Agent 长期决策记录

## 1. 使用规则

本文件记录会长期影响多个任务或构件的决定。任务级实现细节、一次性修复和普通待办不写入此处。

状态取值：

- `proposed`：尚未生效，等待证据或负责人确认。
- `accepted`：当前有效。
- `superseded`：已由新决定替代，保留历史链接。
- `deprecated`：不再适用且没有直接替代。

修改既有决定时不得直接重写历史结论；新增一条决定并用 `supersedes` 指向旧编号。

## 2. 已接受决定

### D-001：以 ISO 14040/14044 为第一阶段标准基线

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：项目目标是高准确、符合标准的通用 LCA；当前本地知识库已包含 ISO 14040/14044 及修正案。
- **决定**：优先建立 ISO 14040/14044 的通用 LCA 主线，覆盖目标与范围、LCI、LCIA、解释、报告和关键审查。
- **影响**：产品碳足迹、EPD/PCR、GHG Protocol 和行业规则作为后续扩展；未建立适用性矩阵与基准前不作覆盖声明。
- **复审条件**：用户目标转向特定声明体系，或项目已达到 M2 并准备扩展标准。

### D-002：使用常青成熟度路线而非固定日期路线

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：研究开发速度、专家资源和 openLCA 环境存在不确定性，固定日期容易与真实完成度脱节。
- **决定**：使用 M0—M4 成熟度、交付物和退出条件推进；日期只记录评审和证据，不代表阶段完成。
- **影响**：阶段可并行准备，但不能跳过依赖或用主观进度替代退出证据。
- **复审条件**：项目进入有明确人员和资源承诺的发布周期。

### D-003：采用风险分级硬门禁

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：文档修改、工具写入和公开比较的后果不同，单一检查集无法兼顾效率与风险。
- **决定**：按 G0—G3 执行递增门禁；零容忍项在所有级别立即生效，高风险结论必须人工审核。
- **影响**：agent 自评不能替代测试或专家结论；关键失败不能被总分平均掉。
- **复审条件**：出现现有分级无法覆盖的新风险类型，或审计发现门禁与事故不匹配。

### D-004：准确性指标先建立基线再收紧

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：当前缺少规模和代表性已确认的金标集，立即指定统一准确率或误差阈值缺乏依据。
- **决定**：零容忍、确定性检查和关键证据覆盖立即采用硬阈值；其他指标先固定基准和定义，再依据实测与专家判断逐版本收紧。
- **影响**：不得用“尚未设阈值”回避测量；每个版本仍须报告基线差异和关键案例结果。
- **复审条件**：基准集稳定、样本量和专家一致性足以支持正式阈值。

### D-005：长期计划聚焦 Harness 核心

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：项目同时包含 GUI、脚本和文件交换，但本轮长期目标是优化 LLM agent 的 Harness 工作流。
- **决定**：路线图覆盖 agent、skill、spec、rule、tool、knowledge、state、observability 和 eval；GUI 与产品能力只在影响 Harness 接口或端到端验证时纳入。
- **影响**：UI 体验、部署和用户管理不作为成熟度退出条件。
- **复审条件**：项目目标升级为完整产品路线或 GUI 成为质量验证的必要边界。

### D-006：构件遵循单一主要职责

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：同一约束在 agent、skill 和 spec 中重复会造成漂移，也让评测难以确定权威来源。
- **决定**：agent 负责角色与编排，skill 负责工作流，spec 负责产物/接口契约，rule 负责跨任务不变量，tool 负责确定性能力与受控副作用。
- **影响**：跨层内容使用引用而非复制；M0 需盘点并处理现有重复和冲突。
- **复审条件**：出现无法由该分层表达的稳定新构件类型。

### D-007：计划必读暂采用约定而非技术强制

- **状态**：`accepted`
- **日期**：2026-07-19
- **背景**：用户选择只在 `docs/plan/` 建立文档，不增加根级 agent 入口或自动校验。
- **决定**：在计划入口规定开发前后的阅读与维护协议，但不新增 `AGENTS.md`、CI 或运行时必读检查。
- **影响**：无法保证所有未来 agent 实际读取；未读导致的偏离需通过评审发现。
- **复审条件**：出现因未读取计划造成的重复设计、门禁遗漏或职责冲突。

## 3. 新决定模板

复制以下模板并使用下一个连续编号：

```markdown
### D-NNN：决定标题

- **状态**：`proposed | accepted | superseded | deprecated`
- **日期**：YYYY-MM-DD
- **supersedes**：D-NNN（如适用）
- **背景**：要解决的问题、现有证据和约束。
- **决定**：采用的方案及明确边界。
- **备选方案**：考虑过但未采用的方案及原因。
- **影响**：对接口、兼容性、测试、迁移和使用者的影响。
- **复审条件**：什么新证据或事件会触发重新评估。
```

决定必须能回到相关标准、测试、评测或实现证据。单纯的模型偏好或无证据直觉不能作为长期约束。
